Git版本控制工具

1、Git 简介

**分布式版本控制系统 （Distributed Version Control System，简称 DVCS） **

在这类系统中，客户端并不只提取最新版本的文件快照， 而是把代码仓库完整地镜像下来，包括完整的历史记录。 这么一来，任何一处协同工作用的服务器发生故障，事后都可以用任何一个镜像出来的本地仓库恢复。 因为每一次的克隆操作，实际上都是一次对代码仓库的完整备份。

不仅如此，许多这类系统都可以指定和若干不同的远端代码仓库进行交互。这样一来，你就可以在同一个项目中，分别和不同工作小组的人相互协作。

集中式和分布式版本控制系统有什么区别呢？

  • 集中式版本控制系统，版本库是集中存放在中央服务器的，最大的毛病就是必须联网才能工作。
  • 分布式版本控制系统根本没有”中央服务器“，每个人的电脑上都是一个完整的版本库。

Git

Git 是一个开源的分布式版本控制系统，旨在快速高效地处理从小型到大型项目的所有内容。【C语言编写】

  • 工作区：就是你在电脑里能看到的目录，用于添加、编辑、修改文件等动作
  • 暂存区：暂存已经修改的文件，最后统一提交到Git仓库中
  • 版本库：最终确定的文件保存到仓库，称为一个新的版本，并且对他人可见

Git 与 SVN 对比

  • 最核心的区别：Git 是分布式的，SVN 是集中式的版本控制系统。

  • Git 把内容按元数据方式存储，而SVN是按文件

    Git 所有的资源控制系统都是把文件的元信息隐藏在一个类似 .svn，.cvs等的文件夹里。

  • Git 分支和 SVN 的分支不同

    分支在 SVN 中一点不特别，就是版本库中的另外的一个目录。

  • Git 没有一个全局的版本号，而 SVN 有

    目前为止这是跟 SVN 相比 Git 缺少的最大的一个特征。

  • Git 的内容完整性要优于 SVN

    Git 的内容存储使用的是SHA-1哈希算法，这能确保代码内容的完整性，确保在遇到磁盘故障和网络问题时降低对版本库的破坏。

2、创建版本库

什么是版本库呢？版本库又名仓库，英文名repository，你可以简单理解成一个目录，这个目录里面的所有文件都可以被Git管理起来，每个文件的修改、删除，Git都能跟踪，以便任何时刻都可以追踪历史，或者在将来某个时刻可以“还原”。

  1. 建立目录

    pwd命令用于显示当前目录。

  1. 设置目录为Git仓库

仓库建好后，会被告知这是一个空的仓库（empty Git repository)，而且当前目录下多了一个.git的目录，这个目录是Git来跟踪管理版本库的，没事千万不要手动修改这个目录里面的文件，不然改乱了，就把Git仓库给破坏了。

  1. 把文件添加到版本库（需要先在仓库中创建文件）

    1. 使用命令git add <file>，注意，可反复多次使用，添加多个文件；

    2. 使用命令git commit -m <message>，完成

      命令执行成功后会告诉你，1 file changed：1个文件被改动（我们新添加的readme.txt文件）；2 insertions：插入了两行内容（readme.txt有两行内容）

注意：Git命令必须在Git仓库目录内执行（git init除外），在仓库目录外执行是没有意义的。

3、版本回退

  • git reset命令返回到任意版本中：git reset –hard 版本id

用HEAD表示当前版本，也就是最新的提交1094adb…（注意我的提交ID和你的肯定不一样），上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100
若是要返回到指定版本，则直接：git reset --hard 版本id
版本号没必要写全，前几位就可以了，Git会自动去找。当然也不能只写前一两位，因为Git可能会找到多个版本号，就无法确定是哪一个了。

  • git log命令查看提交历史，可确定要回退的版本id

    $ git log --pretty=oneline参数–pretty=oneline表示仅显示提交的版本号和信息
  • git reflog命令查看命令历史，可确定要回到未来的哪个版本

4、工作区和暂存区

  • 工作区（指定的文件夹）
    就是你在电脑里能看到的目录，用于添加、编辑、修改文件等动作

  • 版本库（隐藏目录：.git）

    最终确定的文件保存到仓库，称为一个新的版本，并且对他人可见

    • Git为我们自动创建第一个分支master，以及指向master的一个指针叫HEAD
    • git commit命令上就是把暂存区的所有内容提交到当前分支

  • 暂存区/索引（.git/index或者叫stage）

    暂存已经修改的文件，最后统一提交到Git仓库中

    • git add命令是把文件修改添加到暂存区

4、远程仓库

1、添加远程库

本地库推送到远程库

  1. 在GitHub创建新库（同本地库同名）

  2. 使用SSH key公钥关联本地库和GitHub

    在本地库下运行：$ git remote add origin git@github.com:121cai/GitRepository.git
    origin是Git默认的远程库的名字
    移除远程库：git remote remove origin

  3. 把本地库的所有内容推送到远程库上
    $ git push -u origin master
    第一次推送，使用-u参数（之后推送不需要-u参数），Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

  4. GitHub如何配置SSH Key

    • 目的

        待确认：用 git config –global 把user.name和user.email全部重置，email出错可能性不大，但是那个user.name，是github登录之后，鼠标放在右上角的头像那里，显示signed in as XXXX中的XXXX

      2. 检查是否存在SSH Key

        1. 跳转目录：cd ~/.ssh

      3. 显示ssh：ls或者ll，看是否存在 id_rsa 和 id_rsa.pub文件，如果存在，说明已经有SSH Key。（Rsa是算法名称。常用算法有：rsa, dsa。）

        1. 若不存在，则需要生成SSH key：ssh-keygen -t rsa -C "邮箱"

5、分支管理

  • 查看当前分支：git branch，当前分支前面会标一个 * 号

    HEAD严格来说不是指向提交，而是指向master，master才是指向提交的，所以，HEAD指向的就是当前分支。每次提交，master分支都会向前移动一步，这样，随着你不断提交，master分支的线也越来越长。

  • 创建分支：git checkout -b dev或者git swith -c dev命令

    创建新的分支，例如dev时，Git新建了一个指针叫dev，指向master相同的提交，再把HEAD指向dev，就表示当前分支在dev上

    git checkout命令加上-b参数表示创建并切换，相当于以下两条命令：

    从现在开始，对工作区的修改和提交就是针对dev分支了，比如新提交一次后，dev指针往前移动一步，而master指针不变

  • 切换分支git checkout master或者git swith命令
    在其他分支上修改文件并提交后，切换分支后，工作区的文件也会切换后的分支上的文件（未修改前的文件）

  • 合并分支git merge dev（把dev分支的工作成果合并到master分支上）


    合并之后，工作区的数据就变回了dev提交后的数据了。

    注意：若两个分支同时修改提交了数据，合并的话就会出现冲突，需要手动处理冲突

7、GitHub

1、GitHub如何配置SSH Key

用于连接git

目的

  • GitHub配置SSH Key的目的是为了帮助我们在通过git提交代码时，不需要繁琐的验证过程，简化操作流程。

步骤

  1. 设置git的user name和email

    • git设置用户名和邮箱（同gitHub）

      git config命令的–global参数

      • git config –global user.name 用户名
      • git config –global user.email 邮箱

    • eclipse查看/设置

      Preferences》Team》Git>Configuration

  2. 检查是否存在SSH Key

    1. 跳转目录：cd ~/.ssh（实际绝对路径：/c/Users/用户名/.ssh）

    2. 显示ssh：ls 或者 ll，看是否存在 id_rsa 和 id_rsa.pub文件，如果存在，说明已经有SSH Key

      Rsa是算法名称。常用算法有：rsa，dsa。id_rsa.pub是公钥，id_rsa是公钥

    3. 生成SSH key

      • ssh-keygen -t rsa -C 邮箱 【生成密钥对，回车，自动保存密钥对，默认地址：/c/Users/用户名/.ssh】

        注意：输入密钥对密码，全部回车则不设置密码

        如果输入密码则在每次启动Pageant时，需输入一次密码。若当前工作电脑需多人共用，建议加入密码，并在每次操作完毕后关闭Pageant。

      • eclipse创建

        window->preference->general->network connections->ssh2–>Key Management 打开管理页面
        点击Generate RSA KEY，然后点击Save private key，保存文件

    4. 获取SSH key

      cat id_rsa.pub //手动拷贝秘钥（ssh-rsa开头）

  3. GitHub添加SSH Key

    1. GitHub点击用户头像，选择setting
    2. 新建一个SSH Key，取个名字，把之前拷贝的秘钥复制进去

  4. 验证和修改

    1. 测试是否成功配置SSH Key

    2. 修改Http的链接为SSH

      直接修改项目目录下 .git文件夹下的config文件，将地址修改一下就好了
